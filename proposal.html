5  Proposal

Modified

April 19, 2023

Create an interface for defining and sequencing network segments. Model the network as a system of PDEs. Use the finite difference method to find solutions. Produce visualizations that inform the user of the relevant thresholds and margins necessary to provide context for the model output.

The solutions would have to be computed whenever we want results for a different network definition or fluid composition, but if you know the parameters for the fluid components, then it doesnâ€™t take weeks of training a neural net to calculate thermodynamic properties.

If you want to be able to do compositional tracking, you need to be able to model a subsection of the network with a different composition than the rest anyway. Treating each branch (or section between discontinuity points) as separate allows you to do that.

5.1 Interface

If we need a web UI, it makes sense to use React and D3.

5.2 Model

I would choose Julia for the network model. Its speed is comparable to Rust and it has features and packages that make it well suited for scientific and differentiable programming. When it comes to solving differential equations in particular, there are lots of options. Functions can be written in a way that more closely resembles the maths notation that would be familiar to SMEs. Julia can also use libraries from Python, C/Fortran, and other languages and can be called from C.
